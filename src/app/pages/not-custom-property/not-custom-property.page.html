If we turn of all of the --background custom property stuff and use the regular CSS background, we something that works pretty well. The background doesn't scroll up, but the Pull-to-Refresh looks nice without the gap appearing.
The blue line is just to show some of the z-index stuff.
The z-index of the ion-refresher had to be moved up so it's above the bg. This looks fine with just icons. We can see the icon overlap and appear above the content if we pull slowly. This still looks fine even when you notice it and most users will pull quickly.
But this looks bad with text in ion-refresher-content.
